专项训练数据集方面，人类文本训练集完整覆盖 2020 年及之前公开发表的理工科学术论文、核心期刊文献；AI 文本训练集为对应人类文献的 AI 改写、重写样本，整体有效训练样本规模超万篇，全面贴合国内理工科学术写作的真实应用场景。模型实测性能方面，经多轮交叉测试验证，新版模型可稳定、精准地甄别理工科学术场景下的人类原创文本与 AI 生成文本；相较于旧版本，核心检测准确度实现了量级跃升，整体检测能力已正式对标国内主流学术检测平台。目前模型针对 AI 轻度润色类文本的检测精度仍存在优化空间，后续将通过持续的版本迭代完成相关能力的补强。具体可以阅读开发文档

### <span style="color:#1E90FF;">**V 3.0**
修复了一些已知问题

修复了一些图形显示的bug

调整了检测逻辑，现在少于十个字符（不计空格，标点符号，占位符）的段落不再纳入检测范围

调整了计算权重，现在字数更多的长段落占比权重更大，不会受到短段落的影响

使用温度系数调整计算结果，极大减少了极端检测的现象，解决了AI过渡自信的幻觉，增加检测精度

修复了显卡不工作的bug，现在可以正常调用显卡来驱动模型，并在右下角显示硬件状态，如果显卡调用失败会提示用户正在使用CPU进行运算，如果显卡正常则会显示显卡的名称和状态

引入“突发性”（Burstiness）检测：

原理：AI 生成的文本通常句子长度比较平均，节奏单一。而人类写作的句子长短不一，节奏感强。

算法：计算段落中“句子长度的标准差”。如果标准差高（长短句交替），认为这是人类特征，强行扣除 AI 分数。

加大“疑罪从无”力度：

温度系数：从 1.8 提升到 2.0。

指数惩罚：从 2.5 次方提升到 3.5 次方。这样就意味着如果模型只有 80% 的把握，经过计算后会被压到 45% 左右（判定为人类）。只有模型有 95% 以上把握时，才会判定为 AI。

标点丰富度奖励：人类写作往往会使用更丰富的标点（如括号、分号、破折号），而 AI 倾向于使用逗号和句号。如果检测到丰富标点，给予“人类加分”。
