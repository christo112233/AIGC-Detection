# DeepVeri V5.0「学术更新」（已更新）
各位 DeepVeri 的用户、开发者朋友们：

大家好！今日我们正式推出项目上线以来规模最大的全量版本迭代 ——DeepVeri V5.0「学术更新」现已正式发布上线。本次版本完成了核心检测模型、底层代码架构的全面升级重构，并对项目核心发展方向做出了正式调整，现将全部更新详情同步如下：
## 一、核心检测模型全量替换，自研理工科学术专项模型上线
本次 V5.0 版本，已将 DeepVeri 的核心检测模型完整替换为全新自研的理工科学术场景专项模型。该模型专为理工科学术论文检测场景深度定制优化，相关训练数据与实测性能说明如下：

专项训练数据集方面，人类文本训练集完整覆盖 2020 年及之前公开发表的理工科学术论文、核心期刊文献；AI 文本训练集为对应人类文献的 AI 改写、重写样本，整体有效训练样本规模超万篇，全面贴合国内理工科学术写作的真实应用场景。模型实测性能方面，经多轮交叉测试验证，新版模型可稳定、精准地甄别理工科学术场景下的人类原创文本与 AI 生成文本；相较于旧版本，核心检测准确度实现了量级跃升，整体检测能力已正式对标国内主流学术检测平台。目前模型针对 AI 轻度润色类文本的检测精度仍存在优化空间，后续将通过持续的版本迭代完成相关能力的补强。具体可以阅读开发文档
## 二、底层代码结构适配调整，给开发者的提示
底层核心代码已经做了适配工作，现在市面上大部分的模型都可以直接拖入模型文件夹内直接读取，当然，自己训练的也可以使用，并且通过调整开发者控制台的参数来使软件主动适配您的模型
## 三、项目方向正式调整：全面聚焦学术 AIGC 检测
随着 DeepVeri V5.0 版本正式发布及核心检测模型的全量替换，项目已全面向学术文本检测方向倾斜。鉴于项目为个人维护，维护精力有限，后续版本迭代将重点聚焦于提升学术类文本的检测精度，对于普通文本的检测精度，或将有所下降。
若各位开发者有普通文本检测的相关需求，非常欢迎大家拉取项目代码，进行新模型的适配开发。项目底层架构在设计与编写阶段，已充分考虑对不同检测模型的普适性，无需进行大规模代码重构，甚至仅需通过调整开发者控制台相关参数，即可实现新模型的良好适配与稳定运行。

## 📋 前置要求（必看！）
本地部署核心依赖GPU，缺一不可：
1. 电脑需配备NVIDIA GPU（AMD显卡暂不支持）；
2. 已安装CUDA驱动（建议11.7及以上版本）；
3. GPU显存≥2GB；
4. 若无GPU：仅能使用Release版本的CPU轻量化模式（速度较慢，仍可检测）。

无需配置环境，下载即用，推荐使用此方案，Releases提供的都是稳定版，功能可能滞后仓库开源版本，仓库主文件只有更新累计到一定程度时才会更新Relsases,建议阅读更新日志

    双击点开软件后，界面左下角会检测模型是否正确加载，也可手动点击刷新重新加载模型，右下角会检测显卡型号，目前只支持英伟达GPU，AMD GPU尚未测试，应该会出问题，不过可以使用CPU的轻量化模式。

## V5.0更新日志
1、新增token消耗显示（本地部署的模型其实不需要在意token消耗，但是token可以侧面反应硬件的计算量，能直观展示本地显卡或 CPU 刚刚完成了多大的计算负荷，方便排查问题）

2、现在左侧的文本框也可以正确显示输入的文字数量了

3、双击最右侧热力条，会弹出一个新的窗口，在段落较多的情况下热力条会被压缩的过于密集，导致难以精准点击，新的窗口解决了这一问题，并且可以进行筛选，点击卡片依然可以直接联动主界面滚动到对应原文段落

4、现已支持pdf的拖拽导入，但受限于pdf的格式问题，难以正确解码，可能会出现格式错误，换行频繁等问题，所以不建议直接拖拽pdf，优先考虑使用文档格式

5、修复了一些底层的计算问题

6、使用了新模型，对于理工类学术论文有着更高的准确率

7、新增合并段落按钮，现在导入文档可以使用软件自带的段落智能切分功能了，完美的解决了pdf格式错误的问题，当然其他文档也可以使用

8、更新了界面的布局，现在清除按钮已经被单独移动到一侧，并且用了醒目的红色防止误触，更新了各个UI组件使其风格更加统一

9、修复了一些BUG

10、新增开发者控制台
