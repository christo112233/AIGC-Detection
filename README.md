# DeepVeri V5.0「学术更新版」发布预告（已更新）
各位 DeepVeri 的用户、开发者朋友们：
大家好，今天给大家带来项目上线以来最大的一次版本迭代预告 ——DeepVeri V5.0「学术更新」即将正式发布，核心模型、代码架构与项目方向都会有重大调整，相关细节同步给大家：
## 一、核心检测模型全量替换，自研理工科学术专项模型上线
本次 V5.0 版本，会将 DeepVeri 的核心检测模型完整替换为我全新训练的专项模型。该模型完全针对理工科学术论文场景做了深度定制优化，相关训练与实测情况说明如下：
专项训练数据集：人类文本训练集覆盖 2020 年及之前公开发表的理工科学术论文、核心期刊文献；AI 文本训练集为对应人类文献的 AI 改写、重写样本，整体有效训练样本规模超万篇，完全贴合理工科学术写作的真实使用场景。
模型实测表现：经多轮交叉测试验证，新版模型已能稳定、精准地区分理工科学术场景下的人类原创文本与 AI 生成文本；相较于旧版本，核心检测准确度实现了大幅跃升，整体检测能力正式向国内主流学术检测平台靠拢。目前模型针对 AI 轻度润色类文本的检测精度仍有优化空间，后续我会通过持续迭代完成能力补强。
## 二、底层代码结构适配调整，给开发者的提示
由于核心检测模型做了全量替换，本次更新会对项目底层代码结构进行部分重构调整。为了避免大家做无效开发、出现版本适配冲突，麻烦各位参与二次开发的朋友，暂时先不要基于当前版本做修改适配，等 V5.0 版本正式发布上线后，我会同步更新完整的开发文档，届时大家再下载新版包体进行开发即可。
## 三、项目方向正式调整：全面聚焦学术 AIGC 检测
V5.0 版本发布后，DeepVeri 会正式完成核心定位升级，全面聚焦学术领域 AIGC 内容检测。
作为个人开源项目，我的维护精力有限，同时为了兼顾模型轻量化部署的需求，和核心场景的高检测精度，没办法做到全场景兼顾。因此后续的版本迭代，会全面向理工科学术检测场景倾斜：针对符合学术格式规范、包含大量学科专有名词的理工类学术论文，模型检测精度会持续大幅优化，全力为各位同学的学术写作提供可靠的合规校验工具；与之对应，通用普通文本的 AI 检测能力将不再作为核心迭代方向，相关检测精度会有显著下调，还请有通用文本检测需求的用户谅解。
本次 V5.0 版本的正式发布时间，我会在后续的项目更新中同步。感谢大家一直以来对这个开源项目的支持、反馈与贡献，有任何想法和建议，都可以在 issue 里告诉我，我会认真查看每一条留言。

# 🔥 AIGC大模型检测工具（本地部署）
> 免费！中文优化版AIGC检测，使用特调模型，倾向于学术论文检测，精度逼近知网，毕业论文投稿前的AI率初检神器

## 🌟 核心优势
-  中文深度适配：使用特殊训练模型，对中文学术论文检测更精准；
-  完全的图形化操作界面，数据展示一目了然
-  完全免费：无付费门槛，无需登录，本地部署不泄露论文内容；
-  小白友好：提供一键运行版本，无需编程基础，双击即可使用。

## 📋 前置要求（必看！）
本地部署核心依赖GPU，缺一不可：
1. 电脑需配备NVIDIA GPU（AMD显卡暂不支持）；
2. 已安装CUDA驱动（建议11.7及以上版本）；
3. GPU显存≥2GB；
4. 若无GPU：仅能使用Release版本的CPU轻量化模式（速度较慢，仍可检测）。

## 🚀 快速使用（推荐）
无需配置环境，下载即用，推荐使用此方案，Releases提供的都是稳定版，功能可能滞后仓库开源版本，仓库主文件只有更新累计到一定程度时才会更新Relsases,建议阅读更新日志
1. 下载成品包：点击仓库右侧「Releases」，下载最新版本的压缩包（自带完整模型）；
2. 解压运行：将压缩包解压到任意文件夹（建议解压到桌面，方便查找）；
3. 启动检测：双击解压后的「DeepVeri.exe」程序，直接使用：

    双击点开软件后，界面左下角会检测模型是否正确加载，也可手动点击刷新重新加载模型，右下角会检测显卡型号，目前只支持英伟达GPU，AMD GPU尚未测试，应该会出问题，不过可以使用CPU的轻量化模式。

## V5.0更新日志
1、新增token消耗显示（本地部署的模型其实不需要在意token消耗，但是token可以侧面反应硬件的计算量，能直观展示本地显卡或 CPU 刚刚完成了多大的计算负荷，方便排查问题）

2、现在左侧的文本框也可以正确显示输入的文字数量了

3、双击最右侧热力条，会弹出一个新的窗口，在段落较多的情况下热力条会被压缩的过于密集，导致难以精准点击，新的窗口解决了这一问题，并且可以进行筛选，点击卡片依然可以直接联动主界面滚动到对应原文段落

4、现已支持pdf的拖拽导入，但受限于pdf的格式问题，难以正确解码，可能会出现格式错误，换行频繁等问题，所以不建议直接拖拽pdf，优先考虑使用文档格式

5、修复了一些底层的计算问题

6、使用了新模型，对于理工类学术论文有着更高的准确率

7、新增合并段落按钮，现在导入文档可以使用软件自带的段落智能切分功能了，完美的解决了pdf格式错误的问题，当然其他文档也可以使用

8、更新了界面的布局，现在清除按钮已经被单独移动到一侧，并且用了醒目的红色防止误触，更新了各个UI组件使其风格更加统一

9、修复了一些BUG
